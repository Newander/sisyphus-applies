## Project

Local single-user job search tracker. Tracks applications, companies, Gmail messages, and documents. No auth inside the app тАФ single user only. Data stays local (PostgreSQL + local filesystem).

## Architecture

Three processes run together:

- **Frontend** тАФ Next.js 15 / React 19 / TypeScript / Tailwind CSS / shadcn/ui, at `http://localhost:3000`. All API calls go to the backend via `src/lib/api.ts`.
- **Backend** тАФ FastAPI + SQLAlchemy 2 + psycopg3, at `http://127.0.0.1:8000`. Entry point: `backend/main.py`. Routers live in `backend/api/`, business logic in `backend/services/`.
- **Worker** тАФ `backend/worker.py`, runs APScheduler jobs (currently just a heartbeat). Starts independently.

Both backend and worker call `upgrade_database()` on startup, so migrations apply automatically тАФ no manual migration step needed during development.

## Database

PostgreSQL. Core domain models in `backend/models.py`:

- `Company` тЖТ `JobApplication` (many-to-one)
- `JobApplication` has `ApplicationStatus` enum with a defined transition graph (`APPLICATION_STATUS_TRANSITIONS`)
- `ApplicationUpdate` records history for each status change
- `Document` stores file metadata; files live in `JOB_TRACKER_STORAGE_DIR`
- `GmailAccount` + `GmailMessage` for Gmail integration
- `FeatureMemory` for screenshot-based feature tracking

Migrations are in `migrations/` (Alembic). Create a new migration:

```bash
make alembic-revision m="describe the change"
```

## Development commands

### Backend

```bash
# Install dependencies (Linux/Mac тАФ on Windows use scripts/install-backend.ps1)
python -m venv .venv && .venv/bin/pip install -e ".[dev]"

# Run backend
.venv/bin/uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload

# Run worker
.venv/bin/python -m backend.worker

# Lint
.venv/bin/ruff check backend/
.venv/bin/ruff format backend/

# Tests
.venv/bin/pytest backend/tests/
.venv/bin/pytest backend/tests/test_dashboard.py  # single test file
```

### Frontend

```bash
cd frontend
npm install
npm run dev    # dev server at http://localhost:3000
npm run lint   # ESLint
npm run build  # production build
```

### Database migrations (via Makefile)

```bash
make alembic-current          # show current revision
make alembic-upgrade          # upgrade to head
make alembic-revision m="msg" # create autogenerated migration
make alembic-downgrade-one    # roll back one step
```

## Environment

Copy `.env.example` to `.env` and fill in PostgreSQL credentials. Key vars:

| Variable | Default | Notes |
|---|---|---|
| `POSTGRES_*` | localhost/postgres | DB connection |
| `JOB_TRACKER_STORAGE_DIR` | `storage/documents` | Where uploaded files are stored |
| `GOOGLE_CLIENT_SECRET_FILE` | `credentials/client_secret.json` | OAuth secret from Google Cloud Console |
| `GOOGLE_GMAIL_TOKEN_FILE` | `credentials/gmail_token.json` | Written after `connect-gmail` flow |
| `CODEX_CLI_COMMAND` | `codex` | Path to Codex CLI binary |

## Gmail integration

OAuth Desktop App flow with `gmail.readonly` scope. Run `connect-gmail` once to write the token, then use `sync-gmail` for manual sync. The backend also exposes `/api/gmail/sync` for in-app sync triggering.

## Codex bridge

`POST /api/codex/ask` spawns the local Codex CLI, passes a prompt via stdin, returns stdout. Supports `text` mode (direct context) and `url` mode (Playwright scrapes the page first). Prompts live in `backend/prompts/`.

## Key conventions

- `backend/schemas.py` holds Pydantic request/response models (separate from SQLAlchemy `models.py`).
- `backend/config.py` тАФ settings via `pydantic-settings`; `get_settings()` is LRU-cached.
- `backend/db.py` тАФ SQLAlchemy engine + `get_session()` dependency.
- Frontend API client is centralized in `frontend/src/lib/api.ts`; add new endpoints there.
- `ruff` is the only linter/formatter тАФ line length 100, Python 3.12 target.
- No mypy configured yet (noted in docs as future addition after domain model stabilizes).

## OS note

Primary development platform is Windows 10. Scripts in `scripts/` are PowerShell (`.ps1`). On Linux, use the equivalent commands directly.
